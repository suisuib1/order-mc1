# 木材管理系统运行步骤

## 1. 环境要求

- JDK 17
- Maven 3.6+
- Node.js 16+（npm 可用）
- MySQL 8.x

## 2. 数据库准备

数据库配置文件：`ruoyi-admin/src/main/resources/application-druid.yml`

当前默认配置：

- 数据库：`ry-vue`
- 地址：`localhost:3306`
- 用户名：`root`
- 密码：`NewPass123!`

如果本地数据库信息不同，先修改 `ruoyi-admin/src/main/resources/application-druid.yml`。

## 3. 启动后端

在项目根目录执行：

```bash
mvn -pl ruoyi-admin -am package -DskipTests
java -jar ruoyi-admin/target/ruoyi-admin.jar
```

后端默认地址：

- `http://127.0.0.1:8080`

如果只想开发调试，也可以用：

```bash
mvn -pl ruoyi-admin -am spring-boot:run
```

## 4. 启动前端

进入前端目录：

```bash
cd ruoyi-ui
```

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

前端默认访问地址：

- `http://127.0.0.1`

开发环境接口代理配置在：`ruoyi-ui/.env.development`

## 5. 登录系统

默认账号：

- 用户名：`admin`
- 密码：`admin123`

如果开启了验证码，按登录页显示输入即可。

## 6. 常用命令

前端打包：

```bash
cd ruoyi-ui
npm run build:prod
```

后端重新打包：

```bash
mvn -pl ruoyi-admin -am package -DskipTests
```

## 7. 运行异常排查

### 后端起不来

- 检查 `8080` 端口是否被占用
- 检查 MySQL 是否启动
- 检查数据库名、账号、密码是否正确

### 前端起不来

- 检查是否已执行 `npm install`
- 检查 Node.js 版本是否过低
- 检查 `80` 端口是否被占用

### 页面请求 401

- 说明接口存在，但当前未登录或登录态失效
- 重新登录即可

### 页面没有数据或菜单不显示

- 先 `Ctrl + F5` 强刷
- 再退出重新登录一次
